# 业务规则：成员管理

> 规则编号：BR-MM-001
> 关联模块：群成员管理、邀请成员、移除成员、批量移除、申请列表
> 来源：墨刀原型解析 + 用户确认（标注 2、25、36、37、41 + 2026-07-07 确认清单）

---

## 1. 规则描述

成员管理包括邀请成员、移除成员、批量移除成员、审批入群申请。不同角色有不同的操作权限，且部分操作需要二次确认。

## 2. 权限矩阵

| 操作 | 普通用户 | 管理员 | 群主 |
|------|---------|--------|------|
| 查看成员列表 | ✅ | ✅ | ✅ |
| 点击成员头像进入个人主页 | ✅ | ✅ | ✅ |
| 邀请成员 | ❌ | ✅ | ✅ |
| 移除单个成员 | ❌ | ✅ | ✅ |
| 批量移除成员 | ❌ | ✅ | ✅ |
| 审批入群申请 | ❌ | ✅ | ✅ |
| 设置加入需要审核 | ❌ | ✅ | ✅ |

## 3. 邀请成员

### 3.1 入口
- 群管理界面成员网格中的"+"按钮
- 群成员管理-邀请成员页面

### 3.2 规则
- 可选人数上限为群当前剩余可容纳人数
- 已是群成员的用户不可被选择
- 邀请发出后，被邀请人收到站内通知："张三 邀请你加入「群名」群"
- 被邀请人接受邀请后进入群聊

### 3.3 异常处理
| 场景 | 处理规则 |
|------|----------|
| 群已满员 | 提示"圈子人数已达上限"，无法继续邀请 |
| 邀请对象已是群成员 | 选择列表中置灰或隐藏 |
| 邀请被拒绝 | 不发送二次邀请，需重新发起 |

---

## 4. 移除成员

### 4.1 单个移除
- 操作权限：管理员、群主
- 入口：群成员管理-移除成员
- 需二次确认弹窗
- 移除后触发系统消息："李四 已被移出群聊"
- 被移除当事人收到推送："你已被「群名」群移出"

### 4.2 批量移除
- 操作权限：管理员、群主
- 入口：群成员管理-批量移除 或 移除成员页右上角"批量移除"按钮
- 选择成员后统一二次确认
- 批量移除后，每个被移除成员收到相同推送
- 批量移除不影响群主/管理员自身

### 4.3 异常处理
| 场景 | 处理规则 |
|------|----------|
| 移除群主 | 不支持，需先转让群主 |
| 移除管理员 | 支持，移除后自动撤销管理员身份 |
| 移除自己 | 不支持，普通用户通过"退出群聊"实现 |

## 5. 申请加入审批

### 5.1 申请流程
1. 用户点击"立即加入"或申请按钮
2. 填写申请理由（不超过 1000 字，不支持图片）
3. 提交后 Toast 提示"已经成功提交申请"
4. 群主/管理员收到申请通知（红点+站内通知）
5. 审批通过：申请人收到"你已加入「群名」"
6. 审批拒绝：申请人收到"你申请加入「群名」未通过"

### 5.2 默认审核规则
- 用户群：由"加入群需要审核"开关控制
- 赛事群、活动群：默认点击加入需要审核 ⚠️ **待确认**（用户在确认清单中标记为"不确认"）
- 通过报名加入赛事/活动的用户：直接加入，无需审核

### 5.3 审批操作
- 单个同意、单个拒绝
- 一键全部同意、一键全部拒绝
- 有新申请时，入口显示红点

### 5.4 异常处理
| 场景 | 处理规则 |
|------|----------|
| 申请理由超过 1000 字 | 输入框截断或提交时提示"申请理由超过限制" |
| 群已满员 | 申请通过时提示"圈子人数已达上限"，无法加入 |
| 用户已被移除 | 被移除用户再次申请需重新审批 |

## 6. 状态流转

```
用户A
  │ 点击加入
  ▼
填写申请理由（如需审核）
  │
  ▼
提交申请
  │
  ▼
等待审批 ──► 无需审核：直接加入
  │
  ▼
群主/管理员审批
  │
  ├── 同意 ──► 用户加入群聊
  │
  └── 拒绝 ──► 申请失败，通知用户
```

## 8. 测试用例重点

1. 邀请成员时，已是群成员的用户不可选择，群满员时拦截。
2. 单个移除和批量移除的二次确认，移除后系统消息与推送。
3. 管理员被移除后自动撤销管理员身份。
4. 申请理由 1000 字限制，不支持图片。
5. 赛事/活动群默认需审核，报名加入的直接通过。
6. 一键全部同意/拒绝，红点提示，满员时加入失败。
