## 3. 全局设计规范

### 3.1 交互模式

| 模式 | 规则 |
|------|------|
| 操作确认 | 解散群聊、批量移除、删除公告等高风险操作需二次确认弹窗 |
| 状态提示 | Toast 提示时长 2 秒；关键操作如申请提交、移除成功等需即时反馈 |
| 加载/空态/错误态 | 列表支持加载中、空数据、加载失败三种状态；空态提供引导操作 |
| 红点提示 | 入群申请、未读消息、新公告等使用红点/数字角标提醒 |
| 实时更新 | 群聊消息数、未读数、在线状态等实时更新 |

### 3.2 全局状态

| 状态 | 说明 |
|------|------|
| 已登录用户 | 可加入/创建/管理圈子、发言、查看成员 |
| 未登录/游客 | 仅可浏览部分公开圈子信息，不可加入或互动 |
| 已加入 | 可直接进入群聊、接收群消息、参与互动 |
| 未加入 | 显示“立即加入”按钮，点击进入申请/加入流程 |
| 被踢出 | 显示“你已被移出群”，卡片置灰，3 天后自动消失 |
| 群已关闭 | 卡片置灰，不可发言，可查看历史消息 |
| 群已解散 | 卡片置灰，3 天后自动消失 |

### 3.3 通用组件

| 组件 | 说明 |
|------|------|
| 圈子卡片 | 显示圈子名称、成员数、圈主、简介、封面 |
| 消息气泡 | 支持文本、图片、视频、语音、文件、链接、位置、动态卡片、活动卡片 |
| 成员列表 | 头像+昵称+角色标识（群主/管理员/普通成员） |
| 置顶消息区 | 群聊顶部固定区域，最多 5 条消息 + 1 条公告 |
| 公告卡片 | 标题、摘要、发布时间、附件、更多操作菜单 |

## 5. 关键测试关注点（供生成用例时参考）

1. **权限边界**：不同角色能看到/操作的功能差异（如群主才能解散，管理员才能置顶）。
2. **群生命周期状态流转**：正常群 → 关闭 → 重启；正常群 → 解散；自动关闭触发条件。
3. **入群审核**：开关状态、默认规则、报名加入与主动申请的差异。
4. **消息操作**：引用、撤回、转发、多选、举报、置顶、删除的权限与限制。
5. **公告管理**：创建数量上限、附件类型与大小限制、置顶替换、新人必看。
6. **成员管理**：邀请、移除、批量移除、申请审批的流程与异常处理。
7. **搜索与发现**：同城优先、关键字高亮、实时搜索、Tab 切换保留关键字。
8. **系统消息与通知**：入群/退群/踢出/转让/解散等事件的消息触达与可见范围。
